# 管理后台 · 规格说明书

## 一、概述

### 1.1 定位

管理后台是项目管理系统中面向系统管理员的全局配置模块，用于管理成员、权限和标签体系。

### 1.2 访问控制

- 仅 **系统管理员** 可访问管理后台
- 普通成员（`非系统管理员`）无论是否关联某项目，均 **无权访问** 管理后台

### 1.3 入口

顶部导航栏新增 **"管理后台"** 按钮，仅对系统管理员可见。

---

## 二、人员管理

### 2.1 布局

- 页面标题：**人员管理**
- 副标题：**部门共有 N 人**
- 工具栏：搜索框（按姓名模糊搜索）+ **"新增成员"** 按钮（仅管理员）
- 成员列表：卡片式列表，每行一个成员

### 2.2 成员数据结构

```typescript
interface Member {
  id: string               // 系统自增 ID
  name: string             // 成员姓名，必填
  role: 'admin' | 'member' // 系统管理员 | 普通成员

  // 企业微信集成字段
  wecom_user_id: string    // 企业微信 UserId，必填，唯一
  wecom_name: string       // 企业微信昵称
  wecom_avatar: string     // 头像 URL
  mobile: string           // 手机号
  department_id: string    // 企业微信部门 ID（默认 "1"，预留扩展）

  created_at: string
  updated_at: string
}
```

### 2.3 成员列表卡片内容

- 姓名（加粗）
- 角色标签：`系统管理员` / `普通成员`
- 参与项目数：`参与 N 个项目`
- 企业徽标（若已绑定企业微信，显示一个小icon）
- 操作按钮（编辑 / 删除）

### 2.4 功能清单

| 功能 | 权限 | 说明 |
|------|------|------|
| 查看成员列表 | 管理员 | 支持搜索 |
| 新增成员 | 管理员 | 填写上述字段 |
| 编辑成员 | 管理员 | 可编辑姓名、角色、企业微信字段 |
| 删除成员 | 管理员 | 二次确认；成员有关联项目时阻断删除 |
| 绑定企业微信 | 管理员 | 填写 wecom_user_id / mobile |

### 2.5 关联项目展示

- 点击成员卡片 → 展开详情抽屉（Slide-over）
- 详情中展示：该成员参与的项目列表（只读）
- 参与项目的管理 → 在 **项目详情页** 或 **部门面板**，不在管理后台

## 三、权限管理

### 3.1 布局

- 页面标题：**权限管理**
- 子Tab切换：**页面权限** / **项目权限**

### 3.2 页面权限（Page Permissions）

#### 页面列表（固定）

| 页面 Key | 页面名称 |
|----------|---------|
| `department` | 部门级 |
| `project` | 项目级 |
| `admin` | 管理后台 |

#### 功能说明

- 以 **成员** 为维度展示权限矩阵
- 每行一个成员，每列一个页面
- 单元格为 Toggle Switch：开启 = 该成员可访问该页面
- `管理后台` 开关开启时，该成员自动成为系统管理员（role → `admin`）
- `管理后台` 开关关闭时，该成员自动降为普通成员（role → `member`）

> ⚠️ 不允许关闭最后一个 admin 的管理后台权限

### 3.3 项目权限（Project Permissions）

#### 授权方向

**成员 → 项目**（从人出发分配）

#### 功能说明

- 以 **成员** 为维度，展示其可编辑的项目列表
- 每行一个成员，右侧展示该项目列表 + 可管理的项目数
- 点击成员行 → 进入详情抽屉，可增删该项目列表中的项目

#### 权限级别

- `VIEW`：所有部门成员默认可查看所有项目（无需配置）
- `EDIT`：需要显式授权，授权后该成员可编辑该项目

#### 成员详情抽屉内容

- 成员姓名 + 角色
- 可编辑项目列表（Tag 样式，支持移除）
- **"添加项目"** 按钮 → 弹出项目多选框

## 四、标签管理

### 4.1 布局

- 页面标题：**标签管理**
- 工具栏：**"新增标签"** 按钮（所有成员可用）
- 标签列表：网格/列表混合布局

### 4.2 标签数据结构

```typescript
interface Tag {
  id: string
  name: string            // 标签名称，必填，唯一
  color: string           // 预设颜色 key，如 "blue"
  created_at: string
}

type TagColor = 'red' | 'orange' | 'yellow' | 'green' | 'blue' | 'purple' | 'pink' | 'gray'
```

### 4.3 预设色板

每个颜色 key 对应一个固定色值：

| Key | 色值 | 用途 |
|-----|------|------|
| `red` | `#ef4444` | 高优先级 / 紧急 |
| `orange` | `#f97316` | 警告 |
| `yellow` | `#eab308` | 注意 |
| `green` | `#22c55e` | 成功 / 低风险 |
| `blue` | `#3b82f6` | 信息 / 常规 |
| `purple` | `#a855f7` | 设计 |
| `pink` | `#ec4899` | 特殊 |
| `gray` | `#6b7280` | 默认 / 禁用 |

### 4.4 标签在任务/项目上的展示

- 任务/项目详情页中，标签以 **Badge** 形式显示（色块 + 文字）
- **不影响任务卡整体背景色**，仅影响标签 Badge 本身的颜色

### 4.5 功能清单

| 功能 | 所有人 | 管理员 | 说明 |
|------|--------|--------|------|
| 查看标签列表 | ✅ | ✅ | 包含使用量统计 |
| 新增标签 | ✅ | ✅ | 所有成员均可新增 |
| 编辑标签（名称/颜色） | ❌ | ✅ | 仅管理员可编辑 |
| 删除标签 | ❌ | ✅ | 仅管理员可删除 |

> ⚠️ 删除标签时，后端需检查是否有关联任务/项目：
> - 若无关联 → 直接删除
> - 若有关联 → 弹出确认框，列明关联数量："该标签已关联 N 个任务 / M 个项目，删除后关联将自动解除"，确认后删除

### 4.6 标签使用统计

列表中每行显示：`标签名 [色块] | 已关联 X 个任务，Y 个项目`

## 五、技术实现备注

### 5.1 路由结构

```
/admin                    → 管理后台主页（默认跳转至人员管理）
/admin/members            → 人员管理
/admin/permissions/pages  → 页面权限
/admin/permissions/projects → 项目权限
/admin/tags               → 标签管理
```

### 5.2 API 设计（建议）

```
GET    /api/members              → 获取成员列表
POST   /api/members              → 新增成员
PUT    /api/members/:id          → 编辑成员
DELETE /api/members/:id          → 删除成员

GET    /api/permissions/pages    → 获取页面权限矩阵
PUT    /api/permissions/pages    → 更新页面权限

GET    /api/permissions/projects?member_id=   → 获取某成员的 项目权限列表
PUT    /api/permissions/projects/:member_id   → 更新某成员的 项目权限

GET    /api/tags                 → 获取标签列表（含使用量）
POST   /api/tags                 → 新增标签
PUT    /api/tags/:id             → 编辑标签
DELETE /api/tags/:id             → 删除标签（后端检查关联）
```

### 5.3 前端状态管理

- 管理后台数据使用独立的 `adminStore`（Pinia）
- 不与业务数据混用

## 六、视觉/交互约定

- 管理后台整体风格与现有 PM 系统保持一致（共享 CSS 变量）
- 列表页使用卡片列表，详情使用右侧抽屉（Slide-over）而非跳转页
- 删除操作统一使用二次确认对话框
- 表单验证：非空校验 + 唯一性校验（标签名、企业微信 UserId）
